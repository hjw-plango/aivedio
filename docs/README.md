# 项目总览

## 当前结论

本项目是通用型 AI 视频生成平台，支持纪录片、短剧、漫剧等方向。非遗纪录片是第一个深度调优方向，但不是唯一方向。

当前视频生成采用方案 A：

```text
系统生成策划、剧本、分镜、分镜图提示词、即梦视频提示词
→ 用户复制到即梦官网手动生成
→ 用户下载视频并回传
→ 系统记录、评分、绑定到镜头资产
```

不把即梦官网当作 API Provider，不假设网页端能力可以稳定 API 化。

## 自检修正

上一版文档拆得过细，存在三个问题：

- 同一个决策在项目策略、模型策略、路线图、执行流程里重复出现。
- 文档数量偏多，不利于人工审核。
- 非遗纪录片、模型、检索、资产、执行流程被拆散，阅读路径不够直接。

本轮已压缩为 4 个审核入口：

- `docs/README.md`：项目定位、阶段路线、核心取舍。
- `docs/architecture.md`：智能体、模型、检索、资产与执行层。
- `docs/documentary-pilot.md`：非遗纪录片 15 镜 pilot、纪录片美学边界、即梦手动流程。
- `docs/task-log.md`：执行记录和 git 版本。

## 阶段路线

P0 只做非遗纪录片 15 镜 pilot，验证即梦官网能否产出可用纪录片镜头。

P1 做通用前期制片台：项目、剧本、分镜、提示词、资产、执行记录、人工回传。

P2 扩展短剧、漫剧 pilot，并支持 AI 镜头与真实素材混合。

P3 再评估自动化：网页自动化、第三方中转 API 或官方 API。没有稳定入口前，不做全自动视频生成承诺。
