## 用户检查任务

### T1 启动检查

```text
数据库启动
Redis 启动
MinIO 启动
Prisma db push
Next dev server 启动
Worker 启动
Bull Board 启动
```

通过标准：

```text
http://localhost:3000 可访问
http://localhost:3010/admin/queues 可访问
登录或注册可用
设置中心可打开
```

### T2 模型配置检查

在设置中心配置：

```text
文本模型 API
图片模型 API
视频模型 API
语音模型 API
默认画幅比例
并发设置
```

通过标准：

```text
能保存配置
能选择 provider 和 model
任务提交不因余额/计费阻塞
```

### T3 真实内容检查

准备一个小项目，不要一上来用长片。

推荐测试规模：

```text
1 个项目
1 段 800-2000 字故事或剧本
2-4 个主要人物
2-3 个主要场景
3-6 个关键道具
8-16 个分镜面板
```

通过标准：

```text
角色能被识别
场景能被识别
分镜能生成
面板图能生成
同一角色跨镜头基本一致
同一场景跨镜头基本一致
状态变化能通过 appearance 表达
```

### T4 生成质量记录

测试时重点记录：

```text
角色脸是否漂移
服装是否漂移
场景空间是否漂移
道具是否丢失
动作是否符合剧情
镜头语言是否可用
视频是否能承接前后镜头
失败集中在哪个阶段
```

这些记录决定后续是只调提示词，还是需要轻量增强外观/场景锁定能力。
