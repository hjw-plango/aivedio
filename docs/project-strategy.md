# 通用型 AI 视频生成平台修正版方案

## 直接结论

当前项目定位调整为通用型 AI 视频生成平台，支持非遗纪录片、短剧、漫剧等方向。非遗纪录片是首个深度调优方向，不是唯一方向。

当前阶段采用方案 A：系统负责自动生成策划、剧本、分镜、分镜图提示词、即梦视频提示词、执行记录和资产管理；视频生成暂时由用户在即梦官网手动完成，再把结果回传到系统资产库。

不再把“即梦官网可 API 化”作为前提。即梦官网账号当前按人工平台处理，不按正式 API Provider 处理。

第一阶段的重点从“文本流水线”改为“15 镜头纪录片质感验证”。先证明即梦能否生成可用的非遗纪录片镜头，再决定后续自动化深度。

## 项目真实目标

最终目标不是简单生成视频，而是做一个通用 AI 辅助制片系统：

- 自动生成策划案、剧本、旁白、采访提纲、短剧台本、漫剧分镜。
- 自动生成分镜、分镜图提示词、即梦视频提示词。
- 用户可以编辑、重生成、选择候选版本。
- 所有步骤可见、可追踪、可回滚。
- 事实、剧情、人物、资产和版权可追踪。
- 最终产物服务于纪录片、短剧、漫剧等多类型视频生产。

非遗纪录片方向额外强调：

- 文化事实可校验。
- 真实素材和 AI 素材边界明确。
- 传承人、历史影像、采访内容必须尊重真实性和授权。
- 纪录片质感作为首个重点调优目标。

## 修正后的第一阶段

第一阶段不先做完整工程流水线，而是先做非遗纪录片 15 镜 pilot：

```text
3 个非遗主题
每个主题 5 个镜头
总计 15 个镜头提示词
全部人工复制到即梦官网生成
人工打分并记录问题
根据结果判断项目形态
```

三个主题固定为：

- 景德镇制瓷
- 苏绣
- 川剧变脸

这 15 个镜头是纪录片方向的真实信号。结果达到预期，纪录片方向继续向“AI 生成纪录片镜头 + 人类剪辑”推进。结果不稳定，则纪录片方向定位调整为“AI 辅助剪辑非遗素材库 + 分镜提示词系统”。

短剧和漫剧方向不被这 15 镜结果否定，它们会作为后续独立 pilot。

## 结合两个开源项目的取舍

### 继承 AIComicBuilder 的部分

- 继承 `ShotAsset` 思路，用镜头资产承载分镜图、首帧、尾帧、参考图、视频、提示词版本。
- 继承“首帧/尾帧”和“参考图”两种视频生成路径，但第一阶段只用于提示词和手动即梦执行记录。
- 继承 Prompt Registry 的思想，让每类提示词可版本化、可替换、可回滚。

### 继承 waoowaoo 的部分

- 继承 GraphRun / Step / Artifact 的工作流记录思想。
- 继承 TaskEvent 的执行过程可视化思想。
- 继承资产中心，尤其是角色、场景、器具、声音、真实素材的长期复用。
- 继承模型能力配置思想，但第一阶段只有 `manual_jimeng`，不假装有自动 API。

### 我们必须新增的部分

- 非遗事实卡片。
- 纪录片美学规则。
- 真实素材与 AI 素材边界。
- 版权字段。
- 15 镜 pilot 评估体系。
- 手动即梦执行记录。
- 人类专家审核节点。

## 当前系统定位

第一版系统定位为：

```text
通用 AI 视频前期制片台
```

它负责把创意、资料、剧情、文化事实和视觉资产转成可执行的镜头方案。视频生成暂时不是系统自动完成，而是通过即梦官网人工执行。

## 阶段性判断标准

15 镜 pilot 完成后，根据结果做路线判断：

- A 类结果：多数镜头可用，继续做 AI 镜头生成工作流。
- B 类结果：空镜和工艺特写可用，人脸采访不可用，转向“AI 镜头 + 真素材混合纪录片”。
- C 类结果：整体 AI 味严重，转向“分镜、剪辑、素材管理、提示词辅助系统”。

当前默认预判是 B 类：AI 适合生成空镜、工艺特写、纹样动效、历史复原和抽象意象；传承人采访、人脸特写和历史影像必须依赖真实素材。
