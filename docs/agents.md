# 智能体设计修正版

## 直接结论

MVP 不拆 10 个智能体，先压缩成 4 个。拆得太细会增加上下文损耗、调试成本和失败点。

## 4 个核心智能体

### 1. 内容智能体

职责：

- 非遗资料整理。
- 纪录片主题立意。
- 策划案。
- 剧本。
- 旁白。
- 采访提纲。
- 事实卡片引用。

输入：

- 用户主题。
- 参考资料。
- 非遗事实卡片。
- 目标片长。
- 风格要求。

输出：

- 纪录片策划案。
- 分章节大纲。
- 旁白稿。
- 采访问题。
- 事实依据列表。

### 2. 视觉智能体

职责：

- 分镜。
- 摄影语言。
- 美术细节。
- 分镜图提示词。
- 即梦视频提示词。

视觉智能体内部一次性处理导演、摄影、美术和提示词，避免多个智能体互相传递造成风格漂移。

输出：

- Shot 列表。
- Shot 类型：空镜 / 工艺特写 / 人物背影 / 历史复原 / 抽象意象 / 真实素材占位。
- 分镜图提示词。
- 即梦官网提示词。
- 禁止项。
- 需要真拍或授权素材的标记。

### 3. 执行智能体

第一阶段不直接调用即梦 API。执行智能体只负责方案 A：

- 生成可复制到即梦官网的提示词。
- 记录即梦参数。
- 记录生成批次。
- 管理下载后回传的视频文件。
- 绑定结果到 ShotAsset。
- 记录每次重跑原因。

后续可扩展：

- jimeng_rpa_pilot
- volcengine_jimeng
- third_party_proxy

### 4. 质检智能体

职责：

- 检查纪录片真实感。
- 检查 AI 味。
- 检查工艺步骤是否符合事实卡片。
- 检查素材版权字段。
- 检查镜头是否越过“必须真拍”的红线。

质检智能体必须独立，不能和生成智能体合并。

## 智能体执行可视化

每个智能体执行时必须写入事件：

```text
agent_name
step_name
status
input_summary
output_summary
artifact_refs
warnings
created_at
```

UI 上展示为制作流程，而不是单个 loading。

## 当前不做的拆分

以下角色暂不独立成智能体：

- 摄影指导
- 美术指导
- 分镜图提示词工程师
- 视频提示词工程师
- 采访导演

这些都先合并进内容智能体和视觉智能体，等系统跑通后再按真实瓶颈拆分。
