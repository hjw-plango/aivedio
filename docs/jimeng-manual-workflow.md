# 即梦官网手动生成工作流

## 直接结论

当前只采用 `manual_jimeng`。系统不假设即梦官网有可用 API。

## 工作流

```text
系统生成 Shot
→ 系统生成即梦提示词
→ 用户复制提示词到即梦官网
→ 用户按系统建议设置比例、时长、参考图
→ 用户在即梦生成视频
→ 用户下载结果
→ 用户上传回系统
→ 系统绑定到 ShotAsset
→ 系统进入质检与评分
```

## 系统需要记录的信息

```text
shot_id
prompt_version
jimeng_prompt
ratio
duration
reference_images
first_frame
last_frame
manual_run_batch
manual_run_operator
generated_file
score
decision

```

## Prompt 输出格式

每个镜头输出三块：

```text
即梦主提示词
负面约束
人工操作说明
```

示例：

```text
即梦主提示词：
观察式非遗纪录片镜头，景德镇老作坊内，湿润瓷泥在转盘上缓慢成形，匠人的双手只出现手部局部，手指沾满泥水，自然窗光从左侧进入，桌面有泥点和旧工具。手持微距摄影，轻微微抖，焦点在瓷泥边缘和手指之间自然呼吸，低饱和真实色彩。

负面约束：
不要出现清晰正脸，不要广告片质感，不要过度干净，不要塑料质感，不要丝滑轨道运镜，不要奇幻特效，不要错误现代机器。

人工操作说明：
比例 16:9，时长 5-8 秒，优先选择画面稳定但有真实微抖的版本，若出现合成手指或瓷泥形变严重则重跑。
```

## 后续自动化扩展

只在实际确认可用后增加：

- `jimeng_rpa_pilot`：浏览器自动化，内部实验，不作为生产承诺。
- `volcengine_jimeng`：官方 API，只有获得火山引擎即梦 AI API 权限后启用。
- `third_party_proxy`：第三方中转 API，只有完成稳定性和授权验证后启用。

## 当前红线

- 不把网页自动化包装成官方 API。
- 不承诺无人值守批量生成。
- 不在系统里写死火山云密钥或接口。
- 不把即梦官网生成结果自动认定为可发布素材。
