# 测试内容准备清单

## 必须提供

### 1. 一个真实测试项目

建议先准备小规模项目：

```text
项目名
类型：纪录片 / 短剧 / 电影片段
目标语言
目标画幅比例
预计时长
故事或剧本文本
```

推荐文本规模：

```text
800-2000 字
2-4 个主要人物
2-3 个主要场景
3-6 个关键道具
8-16 个分镜面板
```

### 2. 人物资料

每个主要人物提供：

```text
姓名
别名或称呼
身份
年龄段
性别
外貌
发型
服装
性格
人物关系
首次出场状态
后续重要状态变化
```

如果已有参考图，按人物分文件夹准备。

示例：

```text
characters/
  lin_yan/
    face_ref_01.png
    full_body_01.png
  chen_mo/
    face_ref_01.png
```

### 3. 场景资料

每个主要场景提供：

```text
场景名
空间描述
时代或时间段
光线
天气
关键陈设
镜头中常出现的位置
```

如果已有参考图，按场景分文件夹准备。

示例：

```text
locations/
  old_house/
    wide_ref_01.png
    door_ref_01.png
  rainy_street/
    wide_ref_01.png
```

### 4. 道具资料

每个关键道具提供：

```text
名称
外观
材质
大小
属于谁
首次出现在哪
剧情作用
是否会损坏或转移
```

### 5. 模型和 API 信息

需要准备：

```text
文本模型 API Key
图片模型 API Key
视频模型 API Key
语音模型 API Key
希望优先使用的 provider
希望优先使用的 model
```

实际字段以 `waoowaoo-main` 设置中心为准。

## 建议提供

### 1. 质量目标

写清楚这次测试更看重什么：

```text
人物一致性
场景一致性
镜头语言
视频动态
表演情绪
画面真实感
风格统一
```

### 2. 禁止项

写清楚不能出现的内容：

```text
人物不能换脸
服装不能变化
场景不能现代化
不能出现卡通风
不能增加不存在的角色
不能改变剧情顺序
```

### 3. 状态变化表

如果剧情里有人物状态变化，建议提前列出来：

```text
人物    状态名      出现范围        说明
林砚    初始造型    第 1-4 镜       白衬衫，干净，冷静
林砚    雨夜湿身    第 5-8 镜       头发湿，衣服贴身，疲惫
林砚    受伤后      第 9-12 镜      右眉擦伤，外套破损
```

这些状态可以对应 `CharacterAppearance.changeReason`。

## 测试后要记录

每次测试后记录：

```text
使用的文本
使用的模型
角色外观数量
场景参考图数量
生成了多少分镜
成功了多少面板图
失败任务错误信息
最明显的 3 个质量问题
最满意的 3 个结果
```

这些记录用于决定下一轮是否需要改代码。
