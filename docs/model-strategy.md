# 模型策略

## 直接结论

当前阶段只按中转站已提供的最强模型设计，不按成本优化。

核心生成模型：

- 文本、推理、剧本、分镜、质检：`GPT-5.5` 与 `Claude Opus 4.6 / 4.7`
- 生图、分镜图、参考图：`GPT Image 2`
- 视频生成：即梦官网手动执行，系统暂不视为 API Provider

这几个模型足够支撑 P0 和 P1 的核心创作链路，但不足以独立覆盖完整纪录片生产。完整项目还需要检索、知识库、OCR、ASR、字幕、素材理解、版权管理等非生成能力补位。

## 4 个智能体的模型分配

### 内容智能体

主模型：`Claude Opus 4.7`

适用任务：

- 非遗资料消化。
- 长文本结构化。
- 纪录片策划。
- 旁白稿。
- 采访提纲。
- 事实卡片生成。

辅助模型：`GPT-5.5`

适用任务：

- 结构化 JSON 输出。
- 剧本协议校验。
- 多版本改写。
- 内容压缩。

### 视觉智能体

主模型：`GPT-5.5`

适用任务：

- 分镜。
- 镜头语言。
- 摄影参数。
- 即梦视频提示词。
- 分镜图提示词。
- 镜头可剪辑性判断。

辅助模型：`Claude Opus 4.7`

适用任务：

- 风格一致性复核。
- 纪录片语气修正。
- 文化表达克制性检查。

### 执行智能体

主模型：不依赖大模型，优先使用确定性程序。

适用任务：

- 生成即梦官网复制文本。
- 记录手动执行批次。
- 文件命名。
- ShotAsset 绑定。
- 状态流转。

可用模型：`GPT-5.5`

适用任务：

- 执行异常摘要。
- 失败原因归类。
- 自动生成重跑建议。

### 质检智能体

主模型：`GPT-5.5`

适用任务：

- 结构化质检。
- 分镜与事实卡片对齐。
- 镜头评分。
- 失败原因归因。
- 输出可执行修改建议。

辅助模型：`Claude Opus 4.7`

适用任务：

- 文化语境复核。
- 纪录片审美复核。
- 旁白与镜头关系复核。

## GPT Image 2 的使用边界

`GPT Image 2` 适合：

- 分镜图。
- 首帧。
- 尾帧。
- 参考图。
- 器具图。
- 场景图。
- 非真人主体视觉资产。

谨慎使用：

- 传承人正脸。
- 真实采访画面。
- 历史影像。
- 具体人物肖像。

不应使用：

- 冒充真实人物影像。
- 冒充真实档案影像。
- 未经授权复刻具体传承人肖像。

## 仍然需要的辅助能力

仅靠 `GPT-5.5`、`Claude Opus`、`GPT Image 2` 不够覆盖完整系统。还需要以下能力：

### 1. Embedding 与向量检索

用途：

- 非遗知识库检索。
- 事实卡片召回。
- 历史资料片段定位。
- 用户上传资料检索。

要求：

- 支持中文长文本。
- 支持段落级引用。
- 支持元数据过滤。

### 2. Reranker

用途：

- 从多个资料片段里选最相关证据。
- 降低 LLM 乱引用。
- 提高文化事实准确性。

### 3. OCR

用途：

- 识别扫描书籍、展板、地方志、说明牌、老照片文字。

### 4. ASR

用途：

- 真实采访音频转文字。
- 素材字幕提取。
- 后期剪辑检索。

### 5. TTS

用途：

- 临时旁白预览。
- 样片审阅。

正式纪录片旁白仍建议使用真人或授权声音。

### 6. 视频理解模型

用途：

- 回传即梦视频后的自动初筛。
- 检测是否有人脸合成感。
- 检测手部、工具、材料是否异常。
- 生成镜头标签。

当前阶段可以先人工评估，后续再接入。

### 7. 内容安全与版权规则引擎

用途：

- 判断素材来源。
- 检查授权字段。
- 防止把 AI 生成内容标记为真实影像。

## 不需要额外增加的生成模型

在不考虑成本的前提下，P0/P1 暂时不需要再引入更多文本生成模型。模型越多，风格漂移和调试成本越高。

当前文本生成采用双模型互审即可：

```text
Claude Opus 负责长文、文化语境、纪录片表达
GPT-5.5 负责结构化、分镜工程、质检和可执行输出
```

当前生图统一用 `GPT Image 2`，先建立稳定视觉基线。等 pilot 完成后再判断是否需要专门的写实摄影模型或风格微调模型。

## 默认模型策略

```text
内容生成：Claude Opus 4.7
结构化输出：GPT-5.5
分镜与提示词：GPT-5.5
文化语气复核：Claude Opus 4.7
质检评分：GPT-5.5 + Claude Opus 交叉复核
分镜图/首尾帧/参考图：GPT Image 2
视频：manual_jimeng
知识库检索：embedding + reranker
真实采访处理：ASR + 人工校对
```

## 当前项目判断

这些顶级生成模型足够支撑“前期制片 + 分镜提示词 + 分镜图 + 手动即梦视频 pilot”。它们不能替代知识库检索、真实采访处理、版权管理和人工专家审核。

正确路线是：用最强生成模型做创作，用检索和规则系统约束事实，用人工审核守住非遗准确性，用即梦官网验证视频可用性。
