# 产品需求

## P0 目标

把 `waoowaoo-main` 整理成可自用、可测试、可继续打磨的 AI 影视生产项目。

P0 不追求重构，不追求融合多个开源项目，不追求新增完整状态系统。当前重点是验证 `waoowaoo` 原生工作流在真实内容上的生成效果。

## 用户

当前只服务单人自用：

- 本地运行或私有部署。
- 可以注册一个本地账号使用。
- 不做公开 SaaS。
- 不做会员售卖、充值套餐、多人权限。
- 保留用户、任务、资产、模型配置之间的原有依赖。

## 核心工作流需求

### R1 项目创建

必须能创建影视项目，并保存：

- 项目名称。
- 原始故事、小说片段或剧本。
- 目标语言。
- 画幅比例。
- 模型和 API 配置。

### R2 角色一致性

沿用 `waoowaoo` 的角色库：

- 角色名称。
- 角色别名。
- 角色档案。
- 角色介绍。
- 多套外观版本。
- 每套外观的描述、参考图、候选图。

状态变化先用外观版本承载，不新增独立状态表。

示例：

```text
初始造型
换装后
受伤后
雨夜湿身
十年后
战斗状态
```

### R3 场景一致性

沿用 `waoowaoo` 的场景库：

- 场景名称。
- 场景摘要。
- 场景参考图。
- 选中的当前场景图。
- 可用构图槽位。

场景的时间、天气、光线变化先写入场景描述或新增场景图描述，不新增独立场景状态系统。

### R4 分镜生产

必须保留 `waoowaoo` 的多阶段分镜链路：

- 当前 clip 的角色、场景、道具过滤。
- 分镜规划。
- 摄影规则。
- 表演指导。
- 详细镜头和视频 prompt。

分镜阶段优先验证它对角色外观、场景参考和上下文的使用效果。

### R5 面板图和视频生成

必须保留：

- 面板候选图生成。
- `candidateImages`。
- `previousImageUrl`。
- 当前 panel 的角色外观注入。
- 当前 panel 的场景参考注入。
- 视频 prompt 和视频生成链路。

### R6 资产与历史

必须保留：

- 角色外观图历史。
- 场景图历史。
- 面板候选图。
- 视频候选。
- 任务状态。
- 失败记录。

不新增统一资产版本系统，先使用 `waoowaoo` 已有字段。

### R7 计费和会员

P0 不删除计费代码。

当前策略：

```text
BILLING_MODE=OFF
```

保留余额、交易、UsageCost、billing service 源码，避免破坏任务提交和成本统计链路。

## 非目标

P0 不做：

- 旧 Python 项目继续开发。
- AIComicBuilder 代码融合。
- 完整剧本依据系统。
- 完整项目记忆系统。
- ContinuityState 状态机。
- Review Agent。
- 人工评分体系。
- 即梦手动桥。
- 纪录片固定审美 prompt。
- 大规模重命名 `novel-promotion`。
- 删除会员、余额、计费 schema。

## P0 验收标准

P0 完成时必须满足：

- `waoowaoo-main` 可以本地启动。
- 数据库、Redis、MinIO 正常连接。
- 可以注册或登录本地账号。
- 可以配置模型 API Key。
- 可以创建项目并导入故事或剧本。
- 可以生成或维护角色、场景、道具。
- 可以为角色创建多套外观。
- 可以为场景选择参考图。
- 可以生成分镜。
- 可以生成面板图候选。
- 可以继续测试视频、语音、口型同步链路。
- 商业计费不阻塞自用测试。
