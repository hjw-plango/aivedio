# 功能需求

## 角色

P0 只服务一类用户：**内容创作者（也是开发者本人）**。

后续 P1+ 才考虑：策划、编剧、视觉指导、质检、版权管理、专家评审等多角色分工。

## 需求总览

```text
F1  项目与方向管理
F2  研究 Agent
F3  编剧 Agent
F4  分镜 Agent
F5  质检 Agent
F6  Pipeline 编排与可视化
F7  即梦手动桥
F8  资产管理
F9  事实库与知识图谱
F10 版权管理
F11 配置与微调点
F12 系统与可观测性
```

每个需求按 **必须（P0） / 应当（P1） / 可选（P2+）** 标注优先级。

---

## F1 项目与方向管理

### F1.1 创建项目（必须）

- 用户能创建新项目，输入项目名、方向（documentary / drama / comic / general）和 brief。
- 创建后自动加载对应方向的微调配置。
- 项目记录创建时间、当前阶段、关联的所有 GraphRun。

### F1.2 切换项目（应当）

- 用户能在项目列表中切换。
- 切换后所有页面（pipeline、资产、事实库）都切到新项目的上下文。

### F1.3 归档与删除项目（可选）

- 项目可归档（不再展示在主列表，但数据保留）。
- 删除需二次确认，删除后资产文件保留 30 天。

### F1.4 项目配置覆盖（应当）

- 项目级别可覆盖部分微调配置（如改提示词、调阈值），不影响全局默认配置。

---

## F2 研究 Agent

### F2.1 资料输入（必须）

- 支持纯文本粘贴。
- 支持文件上传（txt / md / pdf / 图片）。
- 图片走多模态读取（GPT-5.5 多模态接口），不引入独立 OCR。
- 每条资料保存来源 URL、上传者、上传时间。

### F2.2 FactCard 抽取（必须）

- 从资料抽取结构化事实卡片。
- 字段：`fact_id`, `topic`, `category`（工艺步骤/历史/人物/材料/工具）, `content`, `source_span`, `confidence`。
- 每条 FactCard 必带 SourceSpan（字符级 offset + hash）。
- 用户可手动编辑、合并、拒绝 FactCard。

### F2.3 Entity / Relation 抽取（必须）

- Entity 类型：人物、角色、场景、道具、工艺、镜头、资产。
- Relation 类型：appears_in / uses / before / after / derived_from / requires_real_footage。
- 每个 Entity 和 Relation 带 confidence 和 SourceSpan。
- 冲突项（同名不同义、矛盾关系）进入审核队列。

### F2.4 文化敏感度复核（必须）

- Claude Opus 4.7 二次复核 FactCard。
- 标注潜在文化敏感问题（如错误描述、不当用词）。
- 复核结果作为 FactCard 的 `culture_review` 字段。

### F2.5 资料版本管理（应当）

- 同一资料更新后保留旧版本。
- SourceSpan 按 hash 比对自动判断有效性。
- 失效 span 触发依赖 FactCard 的复核提醒。

---

## F3 编剧 Agent

### F3.1 策划方案生成（必须）

- 输入：FactCard + 项目 brief。
- 输出：策划 JSON（主题立意、目标观众、整体调性、章节结构、核心叙事线）。
- 用户可编辑策划，编辑后可重新生成下游剧本。

### F3.2 剧本生成（必须）

- 输入：策划方案 + FactCard。
- 输出：分场剧本（场次、地点、时间、镜头描述、对白/旁白）。
- 剧本中所有事实性陈述必须引用 FactCard ID。

### F3.3 旁白生成（必须）

- 为每个镜头生成对应旁白文本。
- 旁白与镜头时长匹配（按语速估算，5 秒镜头约 12-15 字）。
- 旁白风格符合纪录片观察式语气，不广告化。

### F3.4 采访提纲生成（应当）

- 为每个传承人生成采访提纲。
- 提纲包含背景了解、核心问题、追问方向。
- 标注哪些问题可由 AI 镜头辅助，哪些必须真拍。

### F3.5 文化语气复核（必须）

- 对剧本和旁白做文化语气检查。
- 标记过度商业化、不当类比、文化误解。

---

## F4 分镜 Agent

### F4.1 分镜表生成（必须）

- 输入：剧本 + FactCard + 美学边界规则。
- 输出：分镜 JSON 列表，每条包含：
  - `shot_id`
  - `scene_id`（关联剧本场次）
  - `shot_type`（空镜/特写/中景/全景/剪影/意象）
  - `subject`（主体）
  - `composition`（构图）
  - `camera_motion`（运镜）
  - `lighting`（光线）
  - `duration_estimate`（建议时长）
  - `narration_ref`（关联旁白）
  - `requires_real_footage`（是否必须真拍）

### F4.2 即梦提示词生成（必须）

- 为每个 `requires_real_footage = false` 的镜头生成即梦提示词。
- 提示词遵循 `configs/documentary/prompts/shot_prompt.md` 模板。
- 包含正向引导 + 负向约束 + 镜头运动 + 风格关键词。
- 提示词长度控制在即梦官网输入限制内。

### F4.3 即梦参数建议（必须）

- 为每个提示词建议：
  - 比例（16:9 / 9:16 / 1:1）
  - 时长（3s / 5s / 10s）
  - 是否使用参考图（首帧 / 尾帧）

### F4.4 分镜图生成（必须）

- GPT Image 2 生成每个镜头的分镜参考图。
- 分镜图作为即梦的参考图候选。
- 分镜图风格保持纪录片质感（不商业广告化）。

### F4.5 语气复核（必须）

- Claude Opus 4.7 检查每条即梦提示词。
- 标记广告化、过度美化、错误工艺描述等问题。

### F4.6 分镜调整（应当）

- 用户可手动调整任意镜头的字段。
- 调整后可单独重生成该镜头的提示词和分镜图，不影响其他镜头。

---

## F5 质检 Agent

### F5.1 事实对齐检查（必须）

- 对剧本、旁白、分镜中的事实陈述与 FactCard 比对。
- 偏差报告包含：偏差描述、关联 FactCard、严重程度（info / warning / error）。

### F5.2 红线检测（必须）

- 检测产物是否触碰红线规则：
  - 是否生成具体传承人正脸
  - 是否冒充真实历史影像
  - 是否生成错误工艺步骤
  - 是否替代专家审核
- 命中红线 → step 标记 rejected，不可自动通过。
- 红线判定需双模型一致才放行。

### F5.3 镜头评分（必须）

- 视频回传后进入人工评分。
- 5 分制 + 失败标签（多选）。
- 评分维度：可用性综合分。
- 失败标签：ai_face / plastic_texture / wrong_craft / ad_style / motion_error / layout_error / irrelevant。
- 系统记录人工评分，不承诺自动视频理解评分。

### F5.4 重跑建议（必须）

- 根据人工填写的失败标签和备注自动给出提示词调整建议。
- 用户可一键应用建议并重新生成。

### F5.5 AI 味归因（应当）

- 基于人工失败标签、文字备注和可选截图，分析哪些元素让人感觉"AI 生成"。
- 输出可执行的改进建议。

### F5.6 人工评分覆盖（必须）

- 用户评分是视频阶段的主评分。
- 如果后续接入视频理解模型，再新增 AI 评分字段，P0 不实现。

## F6 Pipeline 编排与可视化

### F6.1 工作流执行（必须）

- 标准纪录片工作流：研究 → 编剧 → 分镜 → 质检 → 即梦 → 质检。
- 顺序执行，每步完成后默认暂停等待用户确认。
- 用户可设置自动模式（不暂停，全跑完）。

### F6.2 流程图视图（必须）

- 横向或纵向流程图。
- 节点表示 agent，颜色表示状态（pending/running/success/failed/rejected/skipped）。
- 连线表示数据流转。
- 点击节点查看该 step 详情。

### F6.3 步骤详情面板（必须）

- 显示当前 step 的：
  - input 摘要
  - 当前 agent 在做什么（实时事件流）
  - 已产出的 artifact 列表
  - 警告和错误
  - 模型调用记录（哪个模型、token 用量、耗时）

### F6.4 可见性三档切换（必须）

- 全局开关：detail / summary / hidden。
- 单 agent 开关：覆盖全局设置。
- 切换不影响数据落库，只影响展示。
- 任何时候可切回 detail 查看历史 step 的完整事件。

### F6.5 进度摘要折叠（必须）

- agent 的 `progress_note` 默认折叠。
- 点击展开查看可公开的推理摘要、决策依据和进度说明。
- 不保存、不展示模型原始思维链。
- summary 模式下 `progress_note` 隐藏，detail 模式下默认折叠。

### F6.6 重跑（必须）

- 任意 step 可触发重跑。
- 重跑生成新 step，parent_step_id 指向原 step。
- 重跑前可修改 input。

### F6.7 跳过（应当）

- 用户可手动跳过某 step（标记原因）。
- 跳过的 step 状态为 skipped，不影响下游执行。

### F6.8 暂停 / 继续（必须）

- 任意时刻可暂停整个 GraphRun。
- 暂停后可修改 input，再继续。

### F6.9 历史 GraphRun 查看（必须）

- 项目页可查看所有历史 GraphRun。
- 每个 GraphRun 可回放完整事件流。

## F7 即梦手动桥

### F7.1 提示词复制（必须）

- 每个镜头一张复制卡片，显示：
  - 即梦提示词（一键复制）
  - 建议参数（比例、时长、参考图）
  - 关联的分镜图（可下载或拖拽到即梦）
- 复制按钮记录复制时间。

### F7.2 跳转到即梦（应当）

- 提供"打开即梦"按钮，新窗口打开即梦官网。
- 不做自动化操作，仅是便利入口。

### F7.3 视频回传（必须）

- 拖拽上传视频文件。
- 上传时选择关联的 shot_id（可批量）。
- 文件保存到 `assets/{project_id}/videos/{shot_id}_{version}.mp4`。
- 上传成功后自动创建 ShotAsset 记录。

### F7.4 上传后触发质检（必须）

- 视频上传完成后进入人工评分面板。
- 用户填写 5 分制评分、失败标签和备注后，可触发质检 agent 生成归因与重跑建议。

### F7.5 即梦设置记录（应当）

- 用户可手动录入本次生成在即梦上使用的设置（实际比例、时长、模型版本等）。
- 用于后续分析哪些参数效果好。

---

## F8 资产管理

### F8.1 资产列表（必须）

- 按项目展示所有 ShotAsset。
- 支持筛选：方向、类型（提示词/图/视频）、状态（draft/accepted/rejected）、镜头类型。
- 列表展示缩略图、关联 shot、版本号、评分。

### F8.2 候选版本管理（必须）

- 每个镜头每阶段最多保留 3 个候选版本。
- 超出时提示用户选择淘汰哪个。
- 版本状态切换：draft → accepted / rejected。

### F8.3 资产预览（必须）

- 文本/JSON：内联展示。
- 图片：点击放大。
- 视频：内联播放器。
- 提供下载按钮。

### F8.4 资产关联查看（应当）

- 点击资产可查看：
  - 关联的 shot
  - 生成此资产的 step（可跳转）
  - 关联的 FactCard（事实依据）

### F8.5 资产删除（必须）

- 资产可删除，但需二次确认。
- 删除后文件移到 `assets/.trash/`，30 天后清理。

### F8.6 文件命名规范（必须）

```text
分镜图：{project}/{shot_id}/storyboard_v{version}.png
即梦视频：{project}/{shot_id}/jimeng_v{version}_score{score}.mp4
参考图：{project}/{shot_id}/reference_v{version}.png
真拍素材：{project}/{shot_id}/footage_v{version}.mp4
```

## F9 事实库与知识图谱

### F9.1 FactCard 列表（必须）

- 按项目展示所有 FactCard。
- 支持筛选：分类、置信度、复核状态。
- 支持搜索（全文）。

### F9.2 FactCard 编辑（必须）

- 用户可编辑 FactCard 内容、分类、置信度。
- 编辑保留历史版本。

### F9.3 知识图谱可视化（应当）

- 图谱视图：节点（Entity）+ 边（Relation）。
- 点击节点查看关联 FactCard。
- 可按 Entity 类型筛选。

### F9.4 冲突审核队列（必须）

- 系统检测到的冲突项（同名不同义、矛盾关系）进入队列。
- 用户审核：合并 / 保留两个 / 标记某个错误。

### F9.5 EvidencePack 查看（应当）

- 每次 agent 生成时使用的证据包可查看。
- 用于追溯"为什么 agent 给出了这个结果"。

### F9.6 查询规划展示（应当）

- 当 agent 检索时，显示查询规划（拆分了哪些子查询、走了哪些路径）。
- detail 模式下展示，summary 模式下折叠。

## F10 版权管理

### F10.1 版权字段录入（必须）

- AI 生成资产：自动填充（source_type, platform, license, creator）。
- 用户上传资产：必填 source_type, license, rights_holder，否则拒绝入库。
- 第三方授权资产：全字段人工填写。

### F10.2 review_status 流转（必须）

- 默认 pending。
- 用户上传：人工确认后改为 reviewed。
- 第三方授权：需有权限的用户改为 cleared 才能进入成片。

### F10.3 成片合规检查（应当）

- 导出成片前检查所有素材的 review_status。
- 任何 pending 或未 cleared 的素材会阻止导出。

### F10.4 版权报告导出（可选）

- 导出项目的版权清单（所有素材 + 来源 + 授权 + 用途）。

## F11 配置与微调点

### F11.1 配置加载（必须）

- 启动时按方向加载 `configs/{direction}/`。
- 配置包含：prompts、rules（红线、美学、镜头类型）、scoring（评分维度、失败标签）。

### F11.2 配置热更新（应当）

- 修改配置文件后，下次 agent 调用自动使用新配置。
- 不需要重启服务。

### F11.3 配置编辑 UI（可选）

- 提供 Web 界面编辑配置。
- 编辑后版本化保存，可回滚。

### F11.4 多方向支持（应当 P1）

- 新增 `configs/drama/`、`configs/comic/` 后，对应方向的项目自动可用。
- 通用底座代码不需修改。

## F12 系统与可观测性

### F12.1 模型调用记录（必须）

- 每次 LLM 调用记录：
  - 模型名
  - 输入 token 数
  - 输出 token 数
  - 耗时
  - 关联的 step_id
- 用于成本和性能分析。

### F12.2 错误重试（必须）

- LLM 接口错误：自动重试 3 次，指数退避。
- LLM 输出格式错误：自动重试 1 次，仍失败进人工队列。
- 重试记录入 step 的 retry_count。

### F12.3 日志与告警（应当）

- 关键事件落日志（error、red_line、cross_check_mismatch）。
- P1 引入告警通道（邮件 / 飞书）。

### F12.4 数据导出（应当）

- 项目级导出：所有 FactCard、剧本、分镜、ShotAsset、StepEvent 打包为 zip。
- 用于备份、迁移、分享审核。

### F12.5 环境配置（必须）

- LLM API key、数据库连接、资产目录等通过环境变量配置。
- `.env.example` 提供模板。

### F12.6 健康检查（应当）

- `/health` 接口返回数据库、LLM 路由、资产目录状态。

## 非功能需求

### NF1 性能

- P0 单人使用，不做并发优化。
- 单次 agent 调用响应时间不做硬性指标，但需在前端实时展示进度，避免用户认为卡死。
- 资产文件单个不超过 500MB（即梦官网默认输出范围内）。

### NF2 可靠性

- 任何 step 失败都不丢数据。
- 所有 LLM 调用结果落库（即使失败的也记录 error）。
- 资产文件丢失时标记 corrupted，不静默吞错。

### NF3 安全

- API key 不入 git，通过环境变量注入。
- 用户上传文件做 MIME 校验。
- 不做用户认证（P0 单人本地）；P1 引入。

### NF4 可维护性

- Agent 协议统一，新增 agent 不改编排引擎。
- 微调点全部走配置文件，新增方向不改代码。
- 数据库 schema 通过 migration 演进。

### NF5 可观测性

- 所有 step 和 event 完整入库，永不删除。
- 默认 detail 模式可见所有过程；用户信任后可切到 hidden。
- 任何隐藏的过程随时可回看。

## 验收清单（P0 整体）

```text
□ 创建项目 → 选择 documentary 方向 → 加载配置成功
□ 上传资料 → 研究 agent 抽取出 FactCard 和 Entity
□ 编剧 agent 生成策划 + 剧本 + 旁白
□ 分镜 agent 生成 15 个镜头 + 即梦提示词 + 分镜图
□ 质检 agent 完成事实对齐、红线检测、评分
□ 复制提示词到即梦 → 生成视频 → 上传回传
□ 视频自动绑定 ShotAsset → 触发质检
□ 前端实时显示每个 agent 的 progress_note、tool_call、artifact
□ 切换 detail / summary / hidden 三档可见性正常
□ 任意 step 可暂停、修改 input、重跑
□ 所有产物（FactCard、剧本、分镜、提示词、视频、人工评分）均落库
□ 所有版权字段按规则填充
□ 完成 15 镜头 pilot，按成功标准判定方向
```
